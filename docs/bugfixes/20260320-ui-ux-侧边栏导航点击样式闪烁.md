# 侧边栏导航点击时样式闪烁问题

## 基本信息

- **日期**: 2026-03-20 16:00:00
- **问题类型**: ui-ux
- **严重等级**: P2-一般
- **涉及模块**: `components/layout/AppSidebar.tsx`
- **相关 Issue**: N/A

## 问题描述

### 现象

点击侧边栏中的"RAG 模式"下的知识库链接或"普通聊天"下的聊天链接时：
- 页面 URL 正常变化
- 但侧边栏中选中项的高亮样式会短暂丢失
- 直到新页面完全加载后样式才恢复
- 控制台无报错信息

### 复现步骤

1. 打开应用，进入任意页面
2. 点击侧边栏"RAG 模式"下的任意知识库（如"产品文档"）
3. 观察侧边栏中该知识库项的高亮样式
4. 或点击"普通聊天"下的任意聊天会话
5. 观察样式变化

## 根因分析

**关键代码位置**: `components/layout/AppSidebar.tsx:125-155`

**修改前的代码**:
```tsx
// RAG 模式分组 - 知识库列表
...mockKnowledgeBases.map((kb) => ({
  key: `kb-${kb.id}`,
  label: (
    <a href={`/knowledge/${kb.id}`} className="block w-full">
      <DatabaseOutlined style={{ marginRight: 8 }} />
      {kb.name}
    </a>
  ),
  group: 'RAG 模式',
})),
```

**原因分析**:

这是一个 **Next.js 客户端导航与 Ant Design Conversations 组件状态同步时序问题**：

1. **`<a href>` 触发导航**：使用 `<a href="...">` 标签时，Next.js 会拦截点击事件并执行客户端路由导航

2. **URL 立即更新**：浏览器地址栏 URL 立即变化，`usePathname()` hook 检测到路径变化

3. **组件重新渲染**：新页面开始加载，React 组件树重新渲染，包括 `AppSidebar`

4. **`activeKey` 计算延迟**：
   - `selectedKey` 依赖于 `pathname` 计算
   - 但在页面切换过程中，`Conversations` 组件的渲染可能先于 `pathname` 的完全更新
   - 导致短时间内 `activeKey` 与实际路由不匹配

5. **样式丢失**：`Conversations` 组件无法正确匹配选中项，高亮样式丢失

6. **页面加载完成后恢复**：当新页面完全加载，`pathname` 稳定后，`activeKey` 正确计算，样式恢复

**核心问题**：`<a>` 标签的原生行为与 React 受控组件的状态更新存在时序差异。

## 修复方案

### 方案描述

将 `<a href>` 替换为 `onClick + router.push`，完全使用 Next.js 的路由 API 进行导航，避免原生链接行为导致的状态同步问题。

### 关键代码变更

**修改前**:
```tsx
// RAG 模式分组
...mockKnowledgeBases.map((kb) => ({
  key: `kb-${kb.id}`,
  label: (
    <a href={`/knowledge/${kb.id}`} className="block w-full">
      <DatabaseOutlined style={{ marginRight: 8 }} />
      {kb.name}
    </a>
  ),
  group: 'RAG 模式',
})),

// 普通聊天分组
...chats.map((chat) => ({
  key: chat.id,
  label: (
    <a href={`/chat/${chat.id}`} className="block w-full">
      <MessageOutlined style={{ marginRight: 8 }} />
      {chat.title}
    </a>
  ),
  group: '普通聊天',
})),
```

**修改后**:
```tsx
// RAG 模式分组
...mockKnowledgeBases.map((kb) => ({
  key: `kb-${kb.id}`,
  label: (
    <div
      onClick={(e) => {
        e.preventDefault()
        router.push(`/knowledge/${kb.id}`)
      }}
      className="flex items-center w-full cursor-pointer"
    >
      <DatabaseOutlined style={{ marginRight: 8 }} />
      {kb.name}
    </div>
  ),
  group: 'RAG 模式',
})),

// 普通聊天分组
...chats.map((chat) => ({
  key: chat.id,
  label: (
    <div
      onClick={(e) => {
        e.preventDefault()
        router.push(`/chat/${chat.id}`)
      }}
      className="flex items-center w-full cursor-pointer"
    >
      <MessageOutlined style={{ marginRight: 8 }} />
      {chat.title}
    </div>
  ),
  group: '普通聊天',
})),
```

## 备注

### 为什么 `<a href>` 会有问题？

在 Next.js 中：
- `<a href="/internal/path">` 会被 Next.js 拦截并执行客户端导航
- 但拦截过程涉及浏览器默认行为与 React 事件的时序竞争
- 使用 `router.push` 或 `<Link>` 组件可以完全由 React 控制导航流程

### 替代方案

也可以使用 Next.js 的 `<Link>` 组件，但需要处理样式传递：

```tsx
import Link from 'next/link'

<Link href={`/knowledge/${kb.id}`} className="flex items-center w-full">
  <DatabaseOutlined style={{ marginRight: 8 }} />
  {kb.name}
</Link>
```

但 `Conversations` 组件的 `label` 属性期望的是 ReactNode，使用 `onClick` 更直接可控。
