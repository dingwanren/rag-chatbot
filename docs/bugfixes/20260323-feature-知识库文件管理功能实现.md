## 功能描述

### 目标

实现知识库文件管理基础功能，包括：
1. 上传 PDF 文件到 Supabase Storage
2. 展示文件列表
3. 删除文件

### 功能范围

- ✅ 文件上传（仅支持 PDF）
- ✅ 文件列表展示
- ✅ 文件删除
- ❌ Embedding 处理（后续实现）
- ❌ Pinecone 向量存储（后续实现）


### 2. Server Actions 实现

**文件**: `app/actions/knowledge-file.ts`

#### uploadFile(formData: FormData)

```typescript
// 关键流程
1. 获取当前用户 (supabase.auth.getUser())
2. 从 formData 提取 file 和 knowledgeBaseId
3. 校验文件类型为 application/pdf
4. 验证知识库属于当前用户
5. 上传到 Supabase Storage (bucket: rag-files)
6. 获取 public URL
7. 写入 knowledge_files 表
8. 返回 { data, error }
```

#### getFiles(knowledgeBaseId: string)

```typescript
// 关键流程
1. 获取当前用户
2. 验证知识库权限
3. 查询文件列表，按 created_at 倒序
4. 返回 { data, error }
```

#### deleteFile(fileId: string)

```typescript
// 关键流程
1. 获取当前用户
2. 查询文件信息
3. 验证知识库权限
4. 从 Storage 删除文件
5. 删除数据库记录
6. 返回 { data, error }
```

## 注意事项

### 1. Storage 权限配置

确保 Supabase Storage 的 RLS 策略允许认证用户上传和删除文件：

```sql
-- 示例策略（根据实际需求调整）
CREATE POLICY "Authenticated users can upload files"
ON storage.objects FOR INSERT
TO authenticated
WITH CHECK (bucket_id = 'rag-files');

CREATE POLICY "Users can delete their own files"
ON storage.objects FOR DELETE
TO authenticated
USING (bucket_id = 'rag-files');
```

### 2. 文件大小限制

当前未在前端设置文件大小限制，建议在后续迭代中添加：

```typescript
const maxSize = 10 * 1024 * 1024 // 10MB
if (file.size > maxSize) {
  return { data: null, error: new Error('文件大小不能超过 10MB') }
}
```

### 3. 错误处理

- 上传失败时显示错误信息
- 删除失败时显示错误信息
- 权限不足时返回相应错误
