# 限额系统错误处理与用户提示优化

## 📋 更新内容

### 1️⃣ 错误类型识别 (`hooks/useChat.ts`)

现在可以识别不同类型的错误并带有类型标识：

```typescript
// 错误类型
- 'limit_exceeded'  // 限额超限 (429)
- 'unauthorized'    // 未授权 (401)
- 'forbidden'       // 禁止访问 (403)
- 'unknown'         // 其他错误
```

### 2️⃣ 友好的错误提示 (`app/(main)/chat/[id]/page.tsx`)

根据不同错误类型显示不同的提示：

#### 限额超限 (429)
```
⚠️ 今日额度已用完
您的今日请求次数或 token 额度已用完，请明天再来或升级账户计划
```

#### 未授权 (401)
```
❌ 请先登录
您需要先登录才能发送消息
[1 秒后跳转到登录页]
```

#### 禁止访问 (403)
```
⚠️ 无权访问
您没有权限执行此操作
```

### 3️⃣ 限额使用情况实时显示 (`components/chat/UsageIndicator.tsx`)

在聊天页面底部显示：

- 📊 用户等级（Free / Pro / Super）
- 📈 请求次数进度条
- 📈 Token 使用进度条
- ⚠️ 额度预警提示（使用超过 90% 时）

#### Super 用户显示
```
⭐ Super | 无限额度
```

#### Free/Pro 用户显示
```
🚀 Pro | 今日剩余：80 次请求
[进度条] 20/100
[进度条] 5000/100000 tokens
```

## 🔧 技术实现

### 错误处理流程

```
用户发送消息
    ↓
/api/chat-stream 检查限额
    ↓
如果超限 → 返回 429 + 错误信息
    ↓
useSendMessage 捕获并解析错误
    ↓
识别错误类型 (limit_exceeded)
    ↓
抛出带有 type 属性的 Error 对象
    ↓
handleSendMessage 捕获错误
    ↓
根据 error.type 显示对应提示
    ↓
用户看到友好的提示信息 ✅
```

### 限额检查逻辑

```typescript
// 1. 估算 token 使用量
const estimatedTokens = Math.ceil(content.length / 4)

// 2. 获取用户等级
const plan = await getUserPlan(userId)

// 3. Super 用户跳过检查
if (plan !== 'super') {
  const limitCheck = await checkAndUpdateLimits(...)
  if (!limitCheck.success) {
    return 429 { error: '今日请求次数已用完' }
  }
}

// 4. 记录 usage
await recordUsage(...)
```
