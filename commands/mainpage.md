# RAG Chatbot 主界面设计需求

## 布局结构

采用左右布局，左侧为功能侧边栏，右侧为聊天主区域。

```
┌──────────────────────────────────────────────────────────────────┐
│  左侧边栏 (260-300px)           │  右侧聊天区域 (自适应)          │
│ ┌─────────────────────────┐    │  ┌───────────────────────────┐  │
│ │ 🏠 Logo / 应用信息      │    │  │  聊天标题                 │  │
│ │                         │    │  │                           │  │
│ │ ➕ 新聊天               │    │  │                           │  │
│ │                         │    │  │   聊天内容区              │  │
│ │ [RAG 选项] *无知识库时  │    │  │   (消息列表/欢迎页)       │  │
│ │                         │    │  │                           │  │
│ │ ▼ 你的聊天 (可折叠)    │    │  │                           │  │
│ │   - 💬 聊天记录 1      │    │  │                           │  │
│ │   - 💬 聊天记录 2      │    │  │                           │  │
│ │   - ...                │    │  │                           │  │
│ │                         │    │  │                           │  │
│ │                         │    │  │                           │  │
│ │                         │    │  │                           │  │
│ │                         │    │  └───────────────────────────┘  │
│ │                         │    │  ┌───────────────────────────┐  │

│ │ 👤 用户信息/设置        │    │                                 │

```

## 功能说明

### 侧边栏区域

| 区域 | 说明 | 交互 |
|------|------|------|
| **Logo/应用信息** | 品牌展示，可点击返回首页 | 点击重置视图 |
| **新聊天** | 创建新的 AI 对话 | 点击创建普通对话，进入聊天视图 |
| **RAG 选项** | 无知识库时的引导入口 | 点击引导用户创建知识库 |
| **RAG 知识库** | 有知识库时展示，可折叠 | 折叠/展开，管理知识库列表 |
| **新建知识库** | 创建新的知识库 | 点击进入知识库创建流程 |
| **单个知识库** | 展示已创建的知识库 | 点击进入知识库详情/管理 |
| **你的聊天** | 普通对话列表，可折叠 | 折叠/展开，展示所有普通对话 |
| **单条聊天** | 历史对话记录 | 点击切换对应对话 |
| **用户信息** | 当前登录用户信息 | 点击展开设置、退出等选项 |

### 聊天区域

| 区域 | 说明 |
|------|------|
| **聊天标题** | 当前对话标题或默认标题 |
| **聊天内容区** | 消息列表，支持流式输出、代码块、Markdown 等 |
| **用户输入区** | 输入框、发送按钮、可选的附件/上传功能 |

## 交互逻辑

### 模式区分

- **普通聊天**：通过「新聊天」或「你的聊天」中的历史记录进入，不使用知识库
- **RAG 模式**：通过「RAG 知识库」中的具体知识库进入，聊天时关联对应知识库

### 侧边栏折叠

- 「RAG 知识库」和「你的聊天」均为可折叠的分组标题
- 折叠状态本地存储，保持用户偏好
- 折叠时仅显示分组图标和标题

### 知识库状态判断

```typescript
// 伪代码逻辑
if (knowledgeBases.length === 0) {
  // 显示 RAG 选项入口
  showRAGEntry();
} else {
  // 显示 RAG 知识库分组
  showRAGGroup();
}
```

## 响应式考虑

- 侧边栏支持收起/展开（移动端可完全隐藏）
- 小屏幕下侧边栏采用抽屉式滑出
- 聊天区域最小宽度保证 320px

## 技术实现建议

### 组件结构

```
src/
├── components/
│   ├── layout/
│   │   ├── AppLayout.tsx          # 主布局容器
│   │   ├── Sidebar/
│   │   │   ├── Sidebar.tsx        # 侧边栏容器
│   │   │   ├── SidebarHeader.tsx  # Logo 区域
│   │   │   ├── NewChatButton.tsx  # 新聊天按钮
│   │   │   ├── RAGSection.tsx     # RAG 区域 (条件渲染)
│   │   │   ├── ChatSection.tsx    # 聊天列表区域
│   │   │   └── SidebarFooter.tsx  # 用户信息区域
│   │   └── ChatArea/
│   │       ├── ChatArea.tsx       # 聊天主区域
│   │       ├── ChatHeader.tsx     # 聊天标题
│   │       ├── MessageList.tsx    # 消息列表
│   │       └── ChatInput.tsx      # 输入区域
│   └── ...
```

### 状态管理

- 侧边栏折叠状态：`localStorage` 持久化
- 知识库列表：全局状态 (Zustand/Context)
- 聊天列表：按类型分组存储 (普通/RAG)
